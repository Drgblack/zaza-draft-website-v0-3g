# Indexability Policy

## Purpose

This document defines the default indexability policy for major Zaza Draft page families. The goal is to make indexation deliberate rather than incidental.

## Core Rules

1. Only canonical destination pages should be indexed or included in sitemaps.
2. Do not use `robots.txt` to deindex public pages.
3. Use `noindex` for low-value public pages that should remain accessible.
4. Use canonical tags when multiple public URLs intentionally represent the same core content.
5. Use redirects, `404`, or `410` for obsolete or alias routes instead of leaving them indexable.
6. Use sitemap inclusion as a quality filter, not as a full export of every routable URL.
7. Query-parameter states should not be treated as separate indexable pages unless they represent stable canonical destinations.

## Enforcement Priorities

### Immediate

- Remove redirect routes from sitemap inclusion.
- Add missing sitemap coverage for true destination pages such as DE blog posts, compare detail pages, and success-story detail pages.
- Define allowlists or qualification rules for matrix, comparison, generated, and expanded families.

### Next

- Add family-level `robots` policies for low-value detail routes such as community threads, resource details, and placeholder doc-like pages.
- Consolidate overlapping proof sections such as `teacher-stories` and `success-stories`.
- Replace synthetic build-time freshness with real content timestamps for sitemap and metadata freshness signals.

## Policy Summary

- Keep canonical commercial, product, hub, and true editorial pages indexed.
- Keep low-value but legitimate public pages accessible without giving them sitemap priority.
- Noindex low-substance detail families until they are mature.
- Never emit redirect or alias URLs as sitemap candidates.
- Treat large programmatic families as qualified inventory, not auto-index inventory.
